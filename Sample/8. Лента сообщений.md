# Настройка безопасности
Основная цель обеспечения безопасности приложения – это
предотвратить несанкционированный доступ пользователей к важным
страницам, а также к страницам других пользователей, если они не
зарегистрированы. То есть, например, пользователь, зная, что есть страница
users.php, может зайти на нее просто введя ее в адресную строку (рисунок 5). 

Решение этой задачи такое: при каждом посещении страницы надо
проверять пользователя. При это либо выдавать ему сообщение с
предупреждением, либо осуществлять редирект на страницу авторизации
(листинг 3)

```
if(!$_SESSION['logged_user']) {
 header('Location: /index.php');
}
```
Далее создадим в проекте папку function, в которую поместим нашу
проверку безопасности – файл security.php

```
<?php
function security(){
if(!$_SESSION['logged_user']) {
 header('Location: ./index.php');
}
else {
 if(!$_SESSION['logged_user']['admin']) {
 header('Location: ./home.php');
 }
}
}
?>

```

Далее мы можем подключить и вызвать эту функции в файле users.php
в самом начале 


# Настройка времени
Каждый раз, когда мы создаем сообщение в базе данных в атрибуте
created_at создается дата и время создания сообщения. Создадим файл
time.php, в котором поместим функцию humanTiming, листинг которой
приведен ниже

```
<?php
function humanTiming ($time)
 {
 $time = time() - $time; // to get the time since that moment
 $time = ($time<1)? 1 : $time;
 $tokens = array (
 31536000 => 'year',
 2592000 => 'month',
 604800 => 'week',
 86400 => 'day',
 3600 => 'hour',
 60 => 'minute',
 1 => 'second'
 );
 foreach ($tokens as $unit => $text) {
 if ($time < $unit) continue;
 $numberOfUnits = floor($time / $unit);
 return $numberOfUnits.' '.$text.(($numberOfUnits>1)?'s':'');
 }

}
?>

```
Для того, что функция заработала так, как нам нужно надо сначала
преобразовать аргумент функции командой strtotime. Теперь вместо вывода
просто по атрибуту created_at мы будем выводить результат работы нашей
функции
